# SustainDev v0.1.0

**Released: 2026-04-26**
**License: Apache-2.0**

A portable, tool-neutral layer for AI-assisted development that reduces token waste and protects code maintainability across Codex, Claude Code, LM Studio, and MCP-enabled tools.

This is the first public release. Small, opinionated, deliberately incomplete — see the roadmap.

## What's In v0.1.0

The hero workflow `idea-to-prepared-task` plus the minimum core needed to make it credible, measurable, and publishable.

### Foundation (Sprint 0)

- Apache-2.0 license, README that answers five questions above the fold.
- `docs/positioning.md` — fair comparison to Cursor Rules, Cline, Aider, Continue.dev, OpenHands, Claude Code, Codex AGENTS.md.
- `docs/sustainability-defined.md` — what *sustainable* actually means here (token cost, dollar cost, environmental footprint, accessibility, longevity).
- Minimum CI from day one: required-files check, link checker, private-name leak check, SKILL.md schema check.

### Hero workflow (Sprint 1)

- `core/skills/idea-to-prepared-task/SKILL.md` — capture a vague idea, produce a structured task brief, queue it for execution at the right time.
- `core/scheduling/templates/scheduled-task.md` — the brief format the workflow produces.
- `core/scheduling/queue/{captured,prework-ready,scheduled,completed}/` — file-based queue lifecycle.
- `scripts/schedule/capture-idea.sh` and `list-queue.sh` — POSIX shell, no dependencies, idempotent.

### Minimum core

- `core/principles/development-principles.md` — 13 condensed principles.
- `core/rules/token-efficiency.md` — token discipline rules.
- `core/rules/maintainability.md` — six condensed dimensions, with a required maintainability impact note format.
- `core/rules/model-routing.md` — objective routing triggers + strong-fit / weak-fit matrix grounded in case-study-01.
- `core/templates/codemap.md` and `core/templates/project-context.md` — fillable templates for project adoption.

### Adapters

- `adapters/claude-code/CLAUDE.md.template` — thin Claude Code adapter, references core, ~56 lines.
- `adapters/codex/AGENTS.md.template` — thin Codex adapter, mirrors the Claude Code shape.
- `adapters/lm-studio/usage.md` and `prework-prompt.md` — LM Studio setup guide and the production prework prompt for the local-prework tier.

### Empirical evidence

- `docs/measurement/methodology.md` — how to measure honestly.
- `docs/measurement/case-study-01.md` — the first case study with real numbers from a Qwen 3.5 9B run on Apple Silicon. Shows that full-document drafting is a weak fit for the local-prework tier (215s wall-clock for 300 words), and identifies which task classes remain strong fits.

### Coordination demonstration

- `sprint1-handoff/` — the multi-tool coordination package used to build Sprint 1. Three AI tools (Claude Code, Codex, Antigravity-Gemini) each authored a non-overlapping subset of the v0.1 deliverables. Documented in `docs/reviews/sprint-01-review.md`.

## What This Release Is Not

- **Not a complete prompt library.** It ships one workflow with one case study. The roadmap covers what's coming.
- **Not tool-specific.** The core is tool-neutral; adapters are thin glue. If you only use one AI tool, you may prefer that tool's native rule format directly.
- **Not a measurement bonanza.** The strong-fit / weak-fit matrix in `core/rules/model-routing.md` has one row backed by a real measurement. The other nine rows are honestly labeled "Theoretical; not yet measured." v0.2+ fills these in.

## How To Try It

```bash
git clone https://github.com/<your-fork>/sustaindev.git
cd sustaindev

# Read the philosophy
less core/principles/development-principles.md
less core/rules/token-efficiency.md

# Try the hero workflow
./scripts/schedule/capture-idea.sh "your first idea"
./scripts/schedule/list-queue.sh
```

For LM Studio setup (optional but recommended for the prework tier), see `adapters/lm-studio/usage.md`.

## Acknowledgements

This release was authored across three AI tools — Claude Code (Anthropic), Codex (OpenAI), and Antigravity (Google's Gemini-powered IDE) — with Cowork (Claude) coordinating and reviewing. The multi-tool process is itself a hypothesis the project is testing; honest accounting of how it went is in `docs/reviews/sprint-01-review.md`.

The measurement that underlies `core/rules/model-routing.md` was produced with **Qwen 3.5 9B** running locally via **LM Studio** on Apple Silicon. The local model didn't ship the deliverable — that's the lesson — but it informed the routing rules every other tool now follows.

## Honest Limitations

Honest about what this v0.1 is not:

- One measurement case study. Most of the routing matrix is theoretical, awaiting v0.2+ measurements.
- Three adapters (Claude Code, Codex, LM Studio). Warp, VS Code, Cursor, Cline, Antigravity wait for v0.2+.
- Antigravity's prose voice across the documents it authored has noticeable drift from the project style anchor. Documented in the sprint review; v0.2 cleanup planned.
- No cross-document consistency automation yet. Stale-codemap detection, adapter sync scripts, and pattern-promotion automation are v0.7 work.
