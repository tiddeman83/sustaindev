# Code of Conduct

## Purpose

SustainDev exists to help developers build software with AI tools in a more efficient, maintainable, and responsible way. The project should be practical, respectful, and accessible to people using different tools, models, editors, and levels of experience.

## Expected Behavior

Participants are expected to:

- Be respectful and direct.
- Discuss technical disagreements with evidence and clear reasoning.
- Keep feedback focused on the work, not the person.
- Assume contributors may use different AI tools and workflows.
- Be careful with private, sensitive, or proprietary information.
- Help keep the project maintainable and understandable.

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment, intimidation, or discriminatory language.
- Personal attacks or hostile behavior.
- Publishing private information without permission.
- Sharing credentials, secrets, or sensitive project data.
- Repeatedly derailing technical discussions.
- Submitting content that intentionally misleads users about safety, model capability, or unattended automation.

## AI And Automation Expectations

Because this project is about AI-assisted development, contributors should be clear about automation risks.

Do not present AI-generated workflows as safe for unattended use unless they include review gates, scope limits, and verification steps.

Do not encourage tools or models to make risky changes without human review, especially for:

- Security-sensitive code.
- Production operations.
- Data migrations.
- Dependency upgrades with broad impact.
- Changes involving private or regulated data.

Contributors should also be honest about model limitations. Local models, in particular, should be presented as draft-prework tools, not as final-authority code editors.

## Reporting

If you notice behavior that violates this code of conduct, open a private report with the repository owner or maintainer. If GitHub private vulnerability reporting or a maintainer contact is configured later, use that channel.

Reports should include:

- What happened.
- Where it happened.
- Who was involved.
- Any relevant links or screenshots.

## Enforcement

Maintainers may take any action they consider appropriate, including:

- Asking for clarification or edits.
- Removing comments, issues, or pull requests.
- Limiting participation.
- Blocking users from the project.

Enforcement should be proportional, documented when appropriate, and focused on keeping the project useful and safe for contributors.
